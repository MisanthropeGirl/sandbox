IWT2 Security Report Prototyping

Assumptions

  • For the purposes of this I am ignoring chrome (title bar, disclaimer etc)
  • There are 4 parts to the report:
    1. Title
    2. Navigation
    3. Tools
    4. Body
  • Ideally load order: Title, Body, Navigation, Tools
  • (1)&(4) and (2)&(3) can (should?) be grouped together

Steps

  1. HTML for the four distinct parts of the page
  2. Basic CSS (horizontal and vertical navigation layouts)
  3. Flexible navigation
  4. Merge flexible navigation with horizontal and vertical navigation layouts
  5. Columns: 1, 2, 3, 4, 5, 6... etc
  6. Merge columns with horizontal and vertical navigation layouts
  7. Content from V2 report (overview page). Concentrating on the HTML, including accessibility, this page has basic wireframe and presentational styling only.
    1. h* elements within caption elements
    2. h* elements within header elements
    3. Using only h1 elements
  8. Adding styling to V7: 8.1, 8.2, 8.3
  9. Examining Chart resizing via a number of CSS and JavaScript options
  10. Putting content together with menu
  11. Tabbing
  12. Full Demo
  13. Captions or Headers?
  14. Images
  15. Nothing to see here
  16. LESS?
  17. Using the M* Font
  18. Trying out JavaScript loaders
    1. require.js
    2. Modernizr

Notes

  • Where CSS for a responsive layout has been added, layout only changes at a width of 800px. This can be seen either by changing the orientation of a tablet or by reducing the width of the browser window.
  • SRRI has (so far) been displayed using CSS to produce a visual scale. This is unusable when you take styling away so for now I'm just showing the number.
  • Glossary links, inline help or any other explanation mechanisms are not included.
