This is the Growth Chart as it appears on the default implementation of the Fund Report. As we know it does not resize to fit its container.

1(a) Here simple CSS1/2 is used to alter the width (with height rescaling proportionately).

img {
    height:auto;
    width:100%;
}

1(b). CSS3 using the CSS Image Values and Replaced Content Module is used to try and alter the width. Sadly it is not yet widely supported.

img {
    object-fit:contain;
}

1(c). CSS3 using the Flexible Box Layout Module which is widely supported.

container {
    display:flex;
}
img {
    flex:auto;
}

2(a). A jQuery attempt get around the problem of (1b) from Anselm Hannemann. So far my testing has turned up a few issues, one of which is Firefox's blocking of CSS files from 3rd parties.

2(b). A further jQuery attempt get around the problem of (1b), this time from Steve Workman. This does resize but only by assigning a fixed width calculated by the script so is similar to (1a) - and it is processor intensive if manually resizing a window.

2(c). Custom JavaScript. The problem with all of the methods above that have worked is that they all simply stretch the existing image - and the result is rather unpleasant - so in this example I alter the 'chs' and 'chmg' parameters of the BP Chart URL as the container changes.
